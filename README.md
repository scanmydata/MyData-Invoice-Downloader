# Timologio Downloader

Μαζική λήψη παραστατικών **myDATA (ΑΑΔΕ)** για πολλαπλούς πελάτες. Καλεί
`RequestDocs` / `RequestTransmittedDocs`, και για κάθε παραστατικό κατεβάζει το
πραγματικό **PDF** από τον πάροχο μέσω του πεδίου `downloadingInvoiceUrl`.
Δείχνει επίσης, μέσω `RequestE3Info`, αν κάθε παραστατικό είναι χαρακτηρισμένο.

---

## Εγκατάσταση (για τον τελικό χρήστη)

Τρέξτε το **`TimologioDownloader-0.1.0-setup.exe`**.

Δεν χρειάζεται Python, ούτε δικαιώματα διαχειριστή — η εγκατάσταση γίνεται στον
λογαριασμό του χρήστη και δεν εμφανίζεται UAC.

### Τρόπος λειτουργίας

Ο installer ρωτά πώς θα χρησιμοποιηθεί ο υπολογιστής:

| Ρόλος | Φάκελος δεδομένων | Πότε |
|---|---|---|
| **Αυτόνομος** | `Έγγραφα\Παραστατικά myDATA` | ένας υπολογιστής, χωρίς δίκτυο |
| **Server** | τοπικός φάκελος, τον μοιράζεστε μετά | το μηχάνημα που κρατά τη βάση |
| **Τερματικό** | `\\SERVER\ΚΟΙΝΟΧΡΗΣΤΟΣ` | οι υπόλοιποι του γραφείου |

**Σειρά για γραφείο με δίκτυο:**

1. Στον server: εγκατάσταση με ρόλο **Server**.
2. Κάντε τον φάκελο δεδομένων κοινόχρηστο με δικαίωμα **ανάγνωσης/εγγραφής**
   (δεξί κλικ → Ιδιότητες → Κοινή χρήση).
3. Στα τερματικά: εγκατάσταση με ρόλο **Τερματικό**, δίνοντας τη διαδρομή
   `\\ΟΝΟΜΑ-SERVER\ΚΟΙΝΟΧΡΗΣΤΟΣ`.

Δώστε **διαδρομή δικτύου**, όχι γράμμα δίσκου (`Z:\`): τα mapped drives δεν
είναι διαθέσιμα σε κάθε συνεδρία.

> **Μόνο ένας υπολογιστής κατεβάζει κάθε φορά.** Αν πατήσετε «Λήψη» ενώ τρέχει
> ήδη αλλού, θα δείτε ποιος την τρέχει. Αυτό είναι σκόπιμο: αλλιώς δύο μηχανήματα
> θα έκαναν την ίδια δουλειά και θα κατέβαζαν τα ίδια PDF. Η προβολή, η ανάλυση
> και οι αναφορές δουλεύουν κανονικά ταυτόχρονα από όλους.

### Οργάνωση φακέλων

```
%LOCALAPPDATA%\Programs\TimologioDownloader\     ← το πρόγραμμα (~137 MB)
    TimologioDownloader.exe
    unins000.exe                                  ← απεγκατάσταση

<φάκελος δεδομένων>\                              ← ό,τι αξίζει backup
    timologio.db          η βάση: πελάτες, κλειδιά, ιστορικό λήψεων
    .enckey               το κλειδί κρυπτογράφησης των credentials
    backups\              αυτόματα αντίγραφα της βάσης
    sync.lock             υπάρχει μόνο όσο τρέχει λήψη
    ΔΙΑΒΑΣΕ ΜΕ.txt
    data\
        802576637\        ανά ΑΦΜ πελάτη
            2026\
                07\       ανά έτος / μήνα έκδοσης
                    ΑΦΟΙ ΛΑΓΟΥ ΧΡΩΜΑΤΑ ΟΕ_800916954_2026-07-15_ΤΔΑ_3949_202,43.pdf
```

Το όνομα κάθε αρχείου είναι:
`ΠΡΟΜΗΘΕΥΤΗΣ_ΑΦΜ_ΗΜΕΡΟΜΗΝΙΑ_ΣΕΙΡΑ_ΑΑ_ΑΞΙΑ.pdf`

Ο φάκελος δεδομένων **δεν διαγράφεται** κατά την απεγκατάσταση.

## Οδηγίες χρήσης

Η ροή ακολουθεί το πλαϊνό μενού: **Πελάτες** (ποιοι) → **Λήψη** (πότε) →
**Παραστατικά** (τι κατέβηκε).

### 1. Εισαγωγή πελατών

**Νέος πελάτης**: γράψτε το ΑΦΜ (μόνο ψηφία) — μόλις συμπληρωθούν τα 9, η
επωνυμία έρχεται **αυτόματα** από το VIES. Μετά συμπληρώνετε τα credentials.
Στο ίδιο παράθυρο υπάρχει και **Εισαγωγή από Excel…** για μαζική εισαγωγή.

Η μορφή του Excel αναγνωρίζεται αυτόματα:

| Μορφή | Τι διαβάζεται |
|---|---|
| «Κωδικοί Υπηρεσιών μέσω Internet» (εκτύπωση ΑΑΔΕ, σε μπλοκ) | η γραμμή *Ηλεκτρονικά Βιβλία Α.Α.Δ.Ε. (myDATA)* |
| «Κωδικοί Υπόχρεων» (πλατύς πίνακας) | οι στήλες `Α.Φ.Μ.`, `Όνομα χρήστη myData`, `Api myData` |

Πριν γραφτεί οτιδήποτε βλέπετε **preview** με το τι θα αλλάξει. Διαβάζονται μόνο
τα πεδία myDATA — οι υπόλοιποι κωδικοί (Taxisnet, ΕΦΚΑ, ΙΚΑ, ΓΕΜΗ) δεν
φορτώνονται ποτέ στη μνήμη.

> **Οι δύο στήλες κλειδιών δεν είναι ίδιες.** Το `Api myData` είναι το κλειδί
> του REST API. Το `Subscription key e-timologio` είναι **άλλο προϊόν** και δεν
> δουλεύει εδώ. Στο δείγμα σας, 42 από τους 153 πελάτες έχουν μόνο το δεύτερο —
> σωστά εμφανίζονται ως «Λείπει κλειδί API».

### 1α. Διαχείριση πελατών

* **Διπλό κλικ** σε πελάτη *χωρίς* κλειδί → ανοίγει το παράθυρο για να το βάλετε.
* **Διπλό κλικ** σε πελάτη *με* κλειδί → τον (απο)επιλέγει για λήψη.
* **Δεξί κλικ** → επεξεργασία, παραστατικά, εκκαθάριση, διαγραφή.
* **Ctrl/Shift + κλικ** → πολλαπλή επιλογή, μετά δεξί κλικ για μαζική ενέργεια.

Η **διαγραφή** σβήνει πελάτη και εγγραφές· τα PDF στον δίσκο μένουν.

### 2. Λήψη

Στη σελίδα **Λήψη** βλέπετε μόνο τους πελάτες **με κλειδί API** και τσεκάρετε
ποιοι σας ενδιαφέρουν (η επιλογή είναι κοινή με τη λίστα «Πελάτες»). Ορίζετε
περίοδο και πατάτε **Έναρξη λήψης**. Χωρίς καμία επιλογή, κατεβάζει για
**όλους** τους διαθέσιμους — η σελίδα το λέει ρητά πριν πατήσετε.

Έτοιμες περίοδοι με ένα κλικ: τρέχων/προηγούμενος μήνας, τρίμηνο, φέτος, πέρσι.
Οι ημερομηνίες έχουν ημερολόγιο και δέχονται μόνο έγκυρες τιμές.

**Είδος:** τσεκάρετε *Έσοδα (εκδοθέντα)*, *Έξοδα (ληφθέντα)* ή και τα δύο. Είναι
δύο ξεχωριστές κλήσεις της ΑΑΔΕ, οπότε ζητώντας μόνο τη μία η λήψη τελειώνει
στον μισό χρόνο.

Η πρόοδος (πελάτης, πλήθος, σφάλματα και η τελευταία ενέργεια) φαίνεται στο κάτω
μέρος **σε όποια σελίδα κι αν βρίσκεστε**. Το αναλυτικό ιστορικό δεν καταλαμβάνει
πια χώρο στην οθόνη — γράφεται στο **αρχείο καταγραφής** (§10).

Το δεύτερο τρέξιμο δεν ξανακατεβάζει τίποτα — η εφαρμογή κρατά ανά πελάτη το
τελευταίο MARK. Αν κοπεί στη μέση, συνεχίζει από εκεί.

### 3. Ανάλυση ανά πελάτη

Επιλέξτε έναν πελάτη → δεξιά εμφανίζονται παραστατικά, **έσοδα/έξοδα**,
χαρακτηρισμός, κάλυψη ημερομηνιών και κορυφαίοι προμηθευτές.

**Κάθε πλακίδιο είναι κουμπί**: κλικ στα «Σφάλματα» ή στα «Αχαρακτήριστα»
ανοίγει τον πίνακα παραστατικών φιλτραρισμένο σε ακριβώς αυτά. **Διπλό κλικ**
σε πελάτη ανοίγει όλα τα παραστατικά του.

> **Έσοδα ή έξοδα;** Ο κανόνας είναι «έσοδο = το εξέδωσε ο πελάτης», όχι η
> κατεύθυνση. Τα παραστατικά τύπου 13.x/14.x (λιανικά έξοδα) τα υποβάλλει ο
> λήπτης, οπότε εμφανίζονται στα «εκδοθέντα» ενώ είναι **έξοδα**.

Κλικ σε **προμηθευτή** ή σε **τύπο παραστατικού** ανοίγει τα παραστατικά
φιλτραρισμένα σε αυτόν.

### 3α. Πίνακας παραστατικών

> **Η προεπιλεγμένη προβολή είναι τα «Αχαρακτήριστα».** Είναι η μόνη εκκρεμότητα
> που απαιτεί δουλειά, οπότε τη βλέπετε αμέσως. Μια μπλε ταινία το λέει ρητά·
> **Καθαρισμός** για όλα.

Ημερομηνία, **Ε/Ξ**, τύπος, αντισυμβαλλόμενος, σειρά, Α/Α, αξίες, χαρακτηρισμός
και κατάσταση λήψης. Δεξιά κάθε γραμμής κουμπί που **ανοίγει το αρχείο** (PDF ή,
όπου δεν υπάρχει PDF παρόχου, το XML) — ανενεργό όταν δεν υπάρχει αρχείο.

**Τα φίλτρα συνδυάζονται.** Είναι τρεις ανεξάρτητοι άξονες στην πρώτη γραμμή —
**Είδος** (έσοδα/έξοδα), **Λήψη** (αν κατέβηκε το PDF) και **Χαρακτηρισμός** —
που ισχύουν ταυτόχρονα. Παραδείγματα:

| Θέλω να δω | Είδος | Λήψη | Χαρακτηρισμός |
|---|---|---|---|
| Τα έξοδα που έχουν αρχείο | Έξοδα | Ελήφθησαν PDF | Όλα |
| Τη λίστα εργασίας μου | Έξοδα | Όλα | Αχαρακτήριστα |
| Ό,τι δεν πέρασε από πάροχο | Όλα | Χωρίς PDF παρόχου | Όλα |

Στη δεύτερη γραμμή: **προμηθευτής**, **τύπος** και **διάστημα** (με ημερολόγιο).
Πιο κάτω, ελεύθερη αναζήτηση. Τα σύνολα στο κάτω μέρος ακολουθούν πάντα την
τρέχουσα προβολή.

**Εξαγωγή σε ZIP**: τσεκάρετε παραστατικά (ή **διπλό κλικ** στη γραμμή) και
πατήστε το κουμπί. Τα αρχεία μπαίνουν **χύμα** στο ZIP — το όνομα καθενός έχει
ήδη προμηθευτή, ημερομηνία και αξία. Χωρίς επιλογή, μπαίνουν όσα δείχνει η
τρέχουσα προβολή. Αν κάποια δεν έχουν PDF παρόχου, η εφαρμογή **ρωτά** αν θέλετε
να μπουν με το XML της ΑΑΔΕ ή να μείνουν έξω.

### 4. Χαρακτηρισμός (RequestE3Info)

| Ένδειξη | Σημασία |
|---|---|
| **Χαρακτηρισμένο** | έχει χαρακτηριστεί στα Ηλεκτρονικά Βιβλία |
| **Αχαρακτήριστο** | η ΑΑΔΕ το αναφέρει ως «ΜΗ ΧΑΡΑΚΤΗΡΙΣΜΕΝΑ ΕΞΟΔΑ» — θέλει δουλειά |
| **Χωρίς στοιχείο E3** | δεν υπόκειται σε χαρακτηρισμό εξόδων (π.χ. δελτία διακίνησης) |

Το φίλτρο **«Με αχαρακτήριστα»** δείχνει μόνο πελάτες που έχουν εκκρεμότητες.

### 4α. Κενά στις ημερομηνίες

Η εφαρμογή θυμάται **ποια διαστήματα έχουν όντως ζητηθεί**. Αν κατεβάσετε
Ιαν–Μάρ και μετά Ιούν, η ανάλυση δείχνει «⚠ Κενό: 01/04 – 31/05» με κουμπί
**Λήψη κενού**.

Αυτό δεν είναι καλλωπισμός. Το RequestDocs φιλτράρει ταυτόχρονα με MARK και με
ημερομηνίες: αν ζητούσατε τον Απρίλιο **μετά** τον Ιούνιο, τα MARK του Απριλίου
θα ήταν μικρότερα από το τελευταίο που είδαμε και η ΑΑΔΕ **δεν θα επέστρεφε
τίποτα** — σιωπηλά. Η εφαρμογή το ανιχνεύει και ζητά ολόκληρη την περίοδο από
την αρχή όταν χρειάζεται.

### 5. Αντίγραφα ασφαλείας

Αυτόματο αντίγραφο της βάσης πριν από **κάθε** import και **κάθε** λήψη·
κρατούνται τα 10 πιο πρόσφατα ανά είδος. **Αντίγραφο** για χειροκίνητο,
**Επαναφορά…** για γυρισμό πίσω (η τρέχουσα βάση κρατιέται ως «pre-restore»,
οπότε είναι αναστρέψιμο).

### 6. Πελάτες χωρίς κλειδί API

Το φίλτρο **«Χωρίς κλειδί API»** στη λίστα πελατών τους δείχνει όλους μαζί. Το
κλειδί «Api myData» εκδίδεται ανά υπόχρεο από το taxisnet (Ηλεκτρονικά Βιβλία
ΑΑΔΕ → Εγγραφή στο myDATA REST API) — η εφαρμογή δεν μπορεί να το δημιουργήσει.

Για λίστα σε αρχείο: `uv run timologio report missing-keys`.

### 7. Εξαγωγή CSV

Επιλέξτε **πρώτα** πελάτη/πελάτες στη λίστα και μετά **Εξαγωγή CSV**. Χωρίς
επιλογή η εφαρμογή σας το ζητά, αντί να εξάγει σιωπηλά τα πάντα. Το προτεινόμενο
όνομα αρχείου περιέχει **ΑΦΜ και επωνυμία** (`παραστατικά 802576637 ΤΟ ΒΑΨΙΜΟ Ε
Ε.csv`)· με πολλούς πελάτες βγαίνει ένα αρχείο ανά πελάτη.

### 8. Εκκαθάριση ληφθέντων

**Εκκαθάριση** σβήνει τα κατεβασμένα παραστατικά και μηδενίζει το ιστορικό
λήψης, ώστε η επόμενη λήψη να τα ξαναφέρει όλα. **Οι πελάτες και τα κλειδιά
μένουν.** Προαιρετικά διαγράφει και τα αρχεία από τον δίσκο (checkbox στο
παράθυρο επιβεβαίωσης). Χωρίς επιλογή πελατών αφορά όλους· με δεξί κλικ σε
επιλεγμένους, μόνο αυτούς.

### 9. Εμφάνιση

* **Φωτεινό θέμα**: διακόπτης στις Ρυθμίσεις του μενού. Προεπιλογή το σκούρο· η
  επιλογή θυμάται. Βάφεται και η γραμμή τίτλου των Windows.
* **Σύμπτυξη μενού**: το κουμπί ☰ πάνω δεξιά στο μενού το μαζεύει σε λωρίδα
  εικονιδίων και δίνει 142 pixel στους πίνακες. Θυμάται κι αυτό.
* **Στήλες**: κάθε πίνακας είναι **resizable** και **reorderable** — σύρετε τα
  όρια ή τις ίδιες τις επικεφαλίδες. Πλάτη, σειρά και ταξινόμηση αποθηκεύονται.
  Η στήλη της επωνυμίας γεμίζει μόνη της ό,τι περισσεύει· μόλις τη σύρετε
  εσείς, το πλάτος γίνεται δικό σας και δεν το ξαναπειράζει.
* **Δεξί panel**: κλειστό όσο δεν υπάρχει επιλεγμένος πελάτης. Ανοίγει
  συρόμενο μόλις επιλέξετε και κλείνει όταν αποεπιλέξετε.
* Ο **επιλεγμένος πελάτης** φαίνεται μόνιμα κάτω δεξιά, στη γραμμή κατάστασης.
* Όσο η εφαρμογή δουλεύει (φόρτωση, εισαγωγή, εξαγωγή, εκκαθάριση) πέφτει
  **πέπλο αναμονής** που μπλοκάρει τα κλικ, ώστε ένα ανυπόμονο διπλό κλικ να μην
  ξεκινήσει την ίδια δουλειά δεύτερη φορά.

### 10. Βοήθεια

* **Ξενάγηση**: 7 βήματα πάνω από την ίδια την εφαρμογή. Ανοίγει αυτόματα την
  πρώτη φορά και είναι πάντα διαθέσιμη από το μενού.
* **Εγχειρίδιο PDF**: το πλήρες εγχειρίδιο χρήσης. Δημιουργείται την πρώτη φορά
  που θα το ζητήσετε και μπαίνει στον φάκελο δεδομένων.
* **Αρχείο καταγραφής**: `<φάκελος δεδομένων>\logs\timologio.log` — αναλυτικό
  ιστορικό με **ημερομηνία και ώρα Ελλάδας**, ό,τι ώρα κι αν δείχνει ο
  υπολογιστής. Κρατούνται τα 5 τελευταία αρχεία των 2 MB. Τα κλειδιά **δεν**
  γράφονται ποτέ εκεί.

## Τι να περιμένετε

Μετρημένα σε πραγματικό πελάτη (6,5 μήνες, 369 παραστατικά):

* **~89%** έχουν `downloadingInvoiceUrl` και κατεβαίνουν ως PDF.
* **~11%** δεν πέρασαν από κανάλι παρόχου και **δεν έχουν PDF**. Γι' αυτά
  αποθηκεύεται το XML του `RequestDocs` (έχει εκδότη, γραμμές, σύνολα).
* **~70%** των παραστατικών έχουν επωνυμία εκδότη από το myDATA. Με το μητρώο
  και το VIES (παρακάτω) φτάνουμε στο **~97%**. Όπου η επωνυμία παραμένει
  άγνωστη, το όνομα του αρχείου ξεκινά από το ΑΦΜ.

### Από πού έρχονται οι επωνυμίες

Το myDATA δίνει το `<issuer><name>` μόνο στο ~70% των παραστατικών, οπότε η
εφαρμογή χτίζει δικό της μητρώο ΑΦΜ→επωνυμία από τέσσερις πηγές. Σε σύγκρουση
κερδίζει η πιο αξιόπιστη:

| Πηγή | Προτεραιότητα |
|---|---|
| χειροκίνητη διόρθωση | ανώτερη |
| επωνυμία πελάτη από το Excel σας | |
| **VIES** (επίσημο μητρώο ΦΠΑ της ΕΕ) | |
| το `<name>` του παραστατικού | κατώτερη |

Το VIES ρωτιέται **μία φορά ανά ΑΦΜ** (~1 δευτ.), και το αποτέλεσμα —επιτυχία ή
αποτυχία— αποθηκεύεται μόνιμα, οπότε δεν ξαναρωτιέται. Δεν χρειάζεται κλειδί.

Για να συμπληρώσετε επωνυμίες εκ των υστέρων και να μετονομαστούν τα αρχεία:

```bash
uv run timologio names      # αναζήτηση στο VIES
uv run timologio rename     # εφαρμογή στα ονόματα αρχείων
```

## Ασφάλεια

* Τα credentials αποθηκεύονται **κρυπτογραφημένα** (Fernet, πρόθεμα `enc:1:`).
  Το ΑΦΜ και η επωνυμία μένουν καθαρά για αναζήτηση.
* Το κλειδί είναι στο `.enckey` δίπλα στη βάση, με περιορισμένα δικαιώματα.
  **Αν χαθεί**, τα credentials δεν διαβάζονται — ξανακάντε import το Excel.
* Τα διαπιστευτήρια ΑΑΔΕ στέλνονται **μόνο** στο `mydatapi.aade.gr`. Τα URL των
  παρόχων είναι capability URLs, οπότε κατεβαίνουν από ξεχωριστό session
  **χωρίς** auth headers.
* Τα logs περνούν από φίλτρο που κόβει κλειδιά.

## Για προγραμματιστές

### Εκτέλεση από τον κώδικα

```bash
uv run --extra gui python -m timologio.gui   # το GUI
uv run --group dev pytest tests/ -q          # 68 tests
```

> Δεν ορίζουμε `[project.gui-scripts]`: το uv γράφει τότε έναν GUI trampoline
> `.exe` και η εγγραφή του αποτυγχάνει με «Access is denied» σε μηχανήματα όπου
> το site-packages είναι κανονικά εγγράψιμο. Το `python -m` δεν χρειάζεται
> launcher. Ο τελικός χρήστης δεν επηρεάζεται — τρέχει το exe του PyInstaller.

### CLI

```bash
uv run timologio import "Κωδικοί_Υπόχρεων.xlsx" --dry-run
uv run timologio import "Κωδικοί_Υπόχρεων.xlsx"
uv run timologio sync --date-from 01/01/2026 --date-to 31/12/2026
uv run timologio sync --no-vies         # χωρίς αναζήτηση επωνυμιών
uv run timologio names                  # συμπλήρωση επωνυμιών από VIES
uv run timologio rename --dry-run       # μετονομασία στο τρέχον σχήμα
uv run timologio analyse 802576637
uv run timologio report missing-keys
```

### Τεχνικές σημειώσεις

**Το `/pdf` είναι υποχρεωτικό.** Το επίσημο *myDATA API Documentation v2.0.2*
(σελ. 31) λέει ότι το `downloadingInvoiceUrl` χωρίς παράμετρο επιστρέφει PDF
(«default παράμετρος»). **Δεν ισχύει**: δοκιμασμένα, Epsilon και Impact
επιστρέφουν HTML σελίδα προβολής. Η εφαρμογή προσθέτει πάντα `/pdf` και
επαληθεύει τα magic bytes `%PDF` πριν γράψει.

**Η απουσία εγγραφής E3 δεν σημαίνει αχαρακτήριστο.** Σημαίνει ότι το
παραστατικό δεν υπόκειται σε χαρακτηρισμό εξόδων. Το «αχαρακτήριστο» το λέει
ρητά η ΑΑΔΕ με `V_Class_Category = «ΜΗ ΧΑΡΑΚΤΗΡΙΣΜΕΝΑ ΕΞΟΔΑ»`.

**Τα Excel της ΑΑΔΕ σπάνε το openpyxl** («could not read stylesheet»), γι' αυτό
τα διαβάζουμε ξεζιπάροντας το xlsx και παρσάροντας το sheet XML.
